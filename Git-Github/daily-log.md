# Day 3 - Git Branching, Pull & Merge Learning

# Git Workflow Commands

## 1. Create and Switch to New Branch

```bash
git checkout -b feature-login
```

Creates a new branch and switches to it.

---

## 2. Check Current Branch

```bash
git branch
```

Displays all branches and shows the current active branch.

---

## 3. Add Changes

```bash
git add .
```

Adds all modified files to staging area.

---

## 4. Commit Changes

```bash
git commit -m "Added login feature"
`````

Saves changes locally in Git.

---

## 5. Pull Latest Main Branch Updates

```bash
git pull origin main
```

Fetches latest changes from main branch into current branch.

---

## 6. Switch to Main Branch

```bash
git checkout main
```

Switches current branch to main.

---

## 7. Merge Feature Branch into Main

```bash
git merge feature-login
```

Merges feature branch changes into main branch.

---

## 8. Push Changes to GitHub

```bash
git push origin main
```

Uploads updated main branch to GitHub repository.

---

# Easy Flow to Remember

```text
Create Branch
   ↓
Make Changes
   ↓
Add
   ↓
Commit
   ↓
Pull Latest Updates
   ↓
Switch to Main
   ↓
Merge
   ↓
Push
```
