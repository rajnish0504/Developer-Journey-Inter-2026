# Day 12 - React Advanced Hooks, API Data Handling & WMS Validation Testing

## What I Learned Today

* Worked on understanding different data handling concepts such as `Header Content-Type`, `Form Data Content-Type`, and sending `Base64` data through APIs.
* Explored how data transfer works using Fetch API and Axios in React applications.
* Practiced API integration tasks including handling API responses, different request methods, and displaying dynamic data on the UI.
* Improved understanding of request headers, payload structures, and API communication flow in frontend applications.
* Performed WMS testing and validation checks across different Admin modules.
* Tested the **Add New Supplier** form validations and identified an issue where the Phone Number field accepted alphabetic and non-numeric values.
* Verified empty rack action accessibility in the Inventory module and validated that racks with empty rack IDs should remain disabled or unclickable.
* Tested the `+New Entry` functionality based on rack availability conditions.
* Identified a validation issue where the system failed to show proper error messages when all racks were full.
* Learned and practiced important React Hooks including `useRef`, `useMemo`, and `useCallback`.
* Understood the purpose, syntax differences, and real-time usage of advanced React Hooks.
* Explored how these hooks help in performance optimization and preventing unnecessary component re-renders.
* Learned how `useRef` stores mutable values without triggering re-rendering.
* Understood how `useMemo` optimizes expensive calculations using memoization.
* Explored how `useCallback` optimizes function rendering and improves component performance in React applications.
