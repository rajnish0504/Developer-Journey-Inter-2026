# Day 7 - GitHub Workflow, JWT Authentication & Redis Concepts

## What I Learned Today

* Worked on GitHub repositories and practiced creating new branches to understand merge conflicts and repository workflows.
* Learned important Git commands such as `git clone`, `git checkout -b`, `git status`, `git add`, `git commit -m`, and `git push`.
* Understood how merge conflicts occur in Git and learned different ways to resolve them properly.
* Installed and configured the complete development environment including Node.js v22, Eclipse IDE, Visual Studio Code, Git, GitHub, GitHub Desktop, MySQL Workbench, MongoDB, and Postman.
* Created a GitHub account and completed the initial setup required for development work.
* Learned the complete JWT token generation process and understood token expiration handling.
* Studied how Redis cache memory is used for token validation and session management.
* Understood the handling of remaining token validity when users logout before token expiration.
* Learned secure handling of expired or invalidated tokens using cached token data and expiration strategies.
* Explored authentication and authorization concepts using JWT-based workflows.
* Learned the fundamentals of cache memory and how Redis improves authentication performance.
* Compared Session-based Authentication with JWT-based Authentication and understood Stateful vs Stateless architecture.
* Revised JWT token creation, validation flow, and secure backend authentication concepts.
* Attended a knowledge transfer session on JWT authentication flow and backend security implementation.
